# Focus Trainer (Web MVP)

一个基于浏览器的**专注力 / 抗干扰 / 反应控制训练工具**。
无需安装，打开即用。

---

## 项目目标

本项目用于快速验证以下假设：

> 用户是否愿意通过网页形式进行短时专注训练，并产生重复使用行为。

核心策略：

* 不依赖视频
* 不依赖后端
* 不依赖账号系统
* 通过最小交互验证真实需求

---

## 功能概览

### 1. 目标识别训练（Go / No-Go）

* 屏幕中央随机出现符号
* 用户只在出现“目标符号”时做出反应（按键 / 点击）
* 训练能力：
  * 注意控制
  * 反应抑制
  * 选择性注意

### 2. 中央凝视抗干扰训练

* 屏幕中央固定目标点
* 周围随机出现干扰元素（符号 / emoji / 文本）
* 中央目标偶尔变化，用户需及时反应

本质：

* 抑制外部干扰
* 维持注意焦点

### 3. 基础反馈系统

* 命中率
* 漏判率
* 误触率
* 平均反应时间

## 核心设计思想

### 1. 配置驱动（而不是写死逻辑）

所有训练通过配置定义：

```js
const config = {
  targetStimuli: ["★"],
  distractorStimuli: ["●", "▲", "?"],
  intervalMin: 600,
  intervalMax: 1800,
  targetProbability: 0.2
};
```

优势：

* 快速新增训练模式
* 避免重复代码
* 易扩展

### 2. Stimulus Engine（核心模块）

核心不是 UI，而是一个“刺激调度器”：

职责：

* 随机生成刺激
* 控制时间间隔
* 判断目标 / 干扰
* 记录用户反应
* 输出训练数据

### 3. 无状态优先

第一版：

* 不存储用户数据
* 不登录
* 不联网

仅验证：

* 是否有人使用
* 是否有人完成训练
* 是否有人重复进入

## 已知局限

1. **无法检测真实注视点**
   * 没有眼动追踪
   * “盯住中心”只能间接验证
2. **不具备医学意义**
   * 仅为行为训练工具
   * 不等价于“训练前额叶”
3. **内容驱动依赖强**
   * 长期留存依赖训练设计
   * 非纯技术问题
